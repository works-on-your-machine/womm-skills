# Epic: Repo Scaffolding & Plugin Config

**Status:** Ready
**Phase:** 0 — Infrastructure
**Depends on:** Nothing
**Blocks:** Everything else

## Goal

Set up the repository so that someone can fork it, clone it to `~/.womm-skills/`, run `claude /plugin install .`, add the golden specs reference to `~/.claude/CLAUDE.md`, and have everything working.

## Deliverables

### `.claude-plugin/marketplace.json`
Plugin manifest that lists all skills. Updated as skills are added.

```json
{
  "name": "womm-skills",
  "owner": {
    "name": "Scott Werner",
    "url": "https://worksonmymachine.ai"
  },
  "metadata": {
    "description": "Self-improving skill suite for building software with AI",
    "version": "1.0.0",
    "repository": "https://github.com/worksonmymachine/womm-skills"
  },
  "plugins": [
    {
      "name": "womm-skills",
      "description": "Self-improving skill suite for building software with AI",
      "source": "./",
      "strict": false,
      "skills": [
        "./skills/design-doc"
      ]
    }
  ]
}
```

### `CLAUDE.md`
Agent instructions for working within this repo. Should cover:
- What this repo is
- How skills are structured (directory + SKILL.md)
- How golden-specs/ works
- Convention: skills under 500 lines, reference material in references/
- How to add a new skill (create directory, add SKILL.md with frontmatter, update marketplace.json)

### `README.md`
User-facing. Sections:
- What this is (one paragraph)
- Installation:
  1. Fork the repo on GitHub
  2. Clone to `~/.womm-skills/`
  3. `claude /plugin install .` (installs skills)
  4. Add to `~/.claude/settings.json`: allow access to `~/.womm-skills/`
  5. Add to `~/.claude/CLAUDE.md`: reference golden specs at `~/.womm-skills/golden-specs/`
- How it works: skills are cached (plugin), golden specs are read directly (no reinstall needed)
- Skill list by category (table)
- How to customize (add golden specs, modify skills)
- How to add skills (directory structure, SKILL.md format)
- Link to website and newsletter
- Alternative install: copy `skills/` into your project's `.claude/skills/`

### `LICENSE`
MIT.

### `CONTRIBUTING.md`
Short. The contribution model is forking, not PRs. Should cover:
- Fork the repo
- Add your skills, golden specs, modifications
- If you want to share back, open an issue describing what you built (not a PR)

### `golden-specs/` directory
Directory with starter specs and a `README.md` explaining:
- What golden specs are
- What format they follow
- That these are read directly from `~/.womm-skills/golden-specs/` (not cached by the plugin system)
- How `~/.claude/CLAUDE.md` references this directory so all projects can access them
- How to add your own (naming convention, format reference, link to `/golden-spec-create`)
- Starter specs ship in Phase 2 but the directory exists from the start

### `skills/` directory
Empty initially (populated in Phase 1), but the directory should exist.

### `.womm-upstream-sync`
Seed file containing the initial commit SHA. When users fork, this marks their starting point. The `upstream-sync` skill reads this to know what's changed upstream since they last synced.

### Two Mechanisms
- **Skills** → plugin system (cached, available everywhere, reinstall for new skills)
- **Golden specs** → flat files at `~/.womm-skills/golden-specs/`, read directly via `~/.claude/CLAUDE.md` reference. No reinstall needed for new golden specs.

### Convention Over Configuration
- Repo location: `~/.womm-skills/`
- Golden specs: `~/.womm-skills/golden-specs/` (read directly)
- Global reference: `~/.claude/CLAUDE.md` points at golden specs
- Upstream remote: `upstream`
- Sync marker: `.womm-upstream-sync`
- Fork metadata: `womm-manifest.json` (generated by `fork-share`)
